Feature Details

Single / Multi-Node Supply Chain Replenishment Planning

This capability covers both single-node and multi-node network configurations. For single-node networks, targeted strategies optimize inventory management and ensure timely replenishment, enhancing overall efficiency with minimal complexity. For more intricate multi-node networks, the approach involves sophisticated coordination and analytics to manage interdependencies effectively.

Advanced Reinforcement Learning Algorithms to Optimize Replenishment Planning

The deployment of advanced artificial intelligence models to optimize supply chain operations includes the use of the Deep Q-Networks (DQN) model for simpler supply chains. This model is renowned for its efficiency and straightforward application, proving effective in environments with predictable outcomes. It optimizes inventory levels through precise value function estimation and ensures process stability using techniques such as experience replay and fixed Q-targets.

For more complex, multi-node supply chain networks, the Proximal Policy Optimization (PPO) model is utilized. PPO effectively navigates the complexities of intricate systems, optimizing policies directly to adapt flexibly to dynamic conditions. Its incremental and safe policy adjustments provide the robustness needed to effectively manage the uncertainties of supply and demand.

Integrated Forecasting

The integrated advanced forecasting capabilities combine the best of both worlds: cutting-edge machine learning algorithms and proven classical statistical methods. This hybrid approach harnesses the predictive power of machine learning, including deep learning models that adapt and learn from vast data sets, alongside time-tested statistical algorithms that provide stability and reliability in forecast accuracy.

Inputs

Designed to handle both single-node and multi-node network inputs, this capability ensures precise inventory management across varying complexities.

For single-node environments, the inputs are outlined below

  • Retailer Inventory Parameters: SKU, Unit Selling Price, Holding Cost per Day, Fixed Ordering Cost per Order, Variable Ordering Cost per Unit, Inventory Capacity, Max Order Quantity per Order, Min Order Quantity per Order, Initial Inventory, Lead Time Days, Stockout Penalty per unit, Overstock Penalty per Unit
  • Historical Sales: SKU, End of the Week Date, Sold Quantity
  • Sales Forecast (if available): SKU, End of the Week Date, Forecast Quantity
  • Order In-Transit (if available): SKU, Order Date, ETA Date, Order Quantity

For multi-node environments, the inputs are outlined below

  • Distributor Center Inventory Parameters: SKU, Holding Cost per Day, Fixed Ordering Cost per Order, Variable Ordering Cost per Unit, Inventory Capacity, Max Order Quantity per Order, Min Order Quantity per Order, Initial Inventory, Lead Time Days
  • Retailer Inventory Parameters: Retailer, SKU, Unit Selling Price, Holding Cost per Day, Fixed Ordering Cost per Order, Variable Ordering Cost per Unit, Inventory Capacity, Max Order Quantity per Order, Min Order Quantity per Order, Initial Inventory, Lead Time Days
  • Retailer Historical Sales: Retailer, SKU, End of the Week Date, Sold Quantity
  • Retailer Sales Forecast (if available): Retailer, SKU, End of the Week Date, Forecast Quantity
  • Retailer Order In-Transit (if available): Retailer, SKU, Order Date, ETA Date, Order Quantity

Configuration

The capability is equipped with a robust set of user-configurable options that empower businesses to tailor their inventory management and forecasting processes to meet specific needs.

  • Flexibility in Forecasts: Choose between user-generated and system-generated forecasts to leverage internal data insights or automated predictive models.
  • Customization of Forecast Length: Users can set the length of the forecast and planning period in weeks to match their operational timelines and business cycles.
  • Adjustable Week Start and End Days: Allows users to set the start and end days of their week, ensuring integration with existing workflows.
  • Order In-Transit: Includes an option to account for order in-transit information in replenishment calculations, improving the accuracy of inventory assessments.
  • Simulation Options: Users can specify the number of simulations to run, ranging from 100 to 100,000, for detailed risk assessment and scenario planning.
  • Maximum Runtime: Set the maximum runtime for these simulations from 1 to 25 hours, providing flexibility to balance computational depth with operational efficiency.

Outputs

The Reinforcement Learning (RL) algorithm provides detailed output metrics for both single-node and multi-node environments, which are critical for informed decision-making in supply chain management. This capability ensures that businesses can maintain optimal stock levels, meet consumer demand efficiently, and minimize costs related to overstock and understock scenarios.

Summary of Outputs:

  • Inventory Start and End Positions: Shows the inventory position at the beginning and end of the period.
  • Demand and Sales Metrics: Includes total demand, units sold, unfulfilled demand, and demand fulfillment rate.
  • Inventory Management: Details orders arriving, in-transit inventory arriving, projected future inventory, future inventory shortfall, and excess inventory.
  • Actions and Adjustments: Records original and adjusted actions taken, along with reasons for adjustments.
  • Performance Metrics: Provides overstock amounts, capacity utilization rate, and an overall reward metric indicating the efficiency of the inventory management.

Technical Architecture

Frontend Application:

  • React App: Manages the user interface, input file upload and validation, configuration settings and outputs.

Backend Services:

  • Node.js Server: Manages API interactions with MySQL database, ensuring data storage and retrieval operations for the application.
  • Flask API Server: Handles the execution of Python jobs that run reinforcement learning algorithms.
